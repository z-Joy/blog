关于VScode插件 Easy WXLESS

该插件的作用是将less文件编译成wxss文件，具体使用方法等可自行查看插件市场

在使用过程中发现一个问题，插件在编译的时候calc会被忽略单位计算结果在输出，导致样式错乱。

eg. width: calc(1000% - 50rpx); => width: calc(50%);

    在查看使用说明后发现该插件是参考的Easy LESS(注：将less文件编译成css文件)这个插件。在将其换成Easy LESS这个插件测试之后发现并没有上述问题。通过查阅文档发现并没有说明怎么将less转为wxss的参数/方法。不过想这wxss和css除了后缀不一样。写法上是相同的。那么应该是有办法解决的。文档中提到有out这个配置参数。于是将其设置成了 ‘./index.wxss’，亲测有效。

    既然Easy WXLESS是参考WX LESS的，那么编译后的效果怎么会不一样呢，（不把这个搞清楚，晚上怎么睡得着！！！）。于是就去看了两个插件的实现，发现基本一致。关键在于less版本的不同。（Easy WXLESS最新版仍是2017年）

    当然看完实现之后，发现上述的解决方法将【out】设置为 “./index.wxss”是错的。设置的这个值刚好避开了作者的所有判断条件。歪打正着的输入了正确的结果。（废弃）

最后总结一下解决方式。共两种

    方法一

    查看代码发现有提供一个配置参数outExt（注：输出文件后缀名），在使用文档是没有提到过该参数的使用。
  • 卸载Easy WXLESS，安装Easy LESS，并在对应的工作区启用该插件
  • cmd + p;打开工作区设置
  • 找到扩展，Easy LESS
  • 设置 “outExt”: “.wxss”即可

    方法二

    将Easy WXLESS的less版本号升级(本人测试时使用的3.9.0，有效)。
  • 升级less版本，升级version版本，重新打包VSIX文件包
  • 打开vscode,添加扩展 => 从 VSIX安装
  • 安装完成后重启即可

建议使用第一种方案。Easy LESS插件的作者仍然在持续维护升级该插件
